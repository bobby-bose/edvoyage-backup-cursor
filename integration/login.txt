Login Integration Plan
======================

Flutter Page: lib/screens/login/sign_in.dart

1. API Endpoint:
   - Use endpoint: /api/auth/login/ (as per backend convention)
   - Add endpoint constant in lib/constants/api_endpoints.dart

2. API Call:
   - Use http package for POST request to login endpoint
   - Send email/username and password as JSON
   - Handle response codes (200: success, 400/401: error)
   - Parse and store JWT/auth token on success

3. State Management:
   - Use provider/riverpod for auth state
   - Show loading indicator during request
   - Show error message on failure (SnackBar/AlertDialog)

4. Debugging:
   - Print debug info in kDebugMode (method, status, response)

5. Null Safety:
   - Check for nulls in response and user input

6. UI:
   - Disable login button while loading
   - Navigate to home/dashboard on success

7. Error Handling:
   - Handle SocketException, TimeoutException, FormatException
   - Show user-friendly error messages

8. Token Storage:
   - Store token securely (e.g., flutter_secure_storage or shared_preferences)

9. Testing:
   - Test with Postman and Django backend

Refer to integration_guidelines.txt for standards.
